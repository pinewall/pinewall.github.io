# LLM 大语言模型基本原理

大语言模型（Large Language Model, LLM）是当前人工智能领域最热门的方向之一，它基于深度学习技术，通过学习海量文本数据，能够理解和生成人类语言。本文介绍 LLM 的基本原理。

## 什么是大语言模型

大语言模型是一个**概率模型**，它学习人类语言的统计规律，能够：

- 给定上文，预测下一个词最可能是什么
- 理解自然语言的含义和语境
- 生成流畅、连贯的文本
- 进行推理、问答、翻译等任务

核心思想：**语言建模 =  next token 预测**

## 基础概念

### 词嵌入（Word Embedding）

将词语转换为向量表示：

- 每个词映射到一个高维向量（通常 512 ~ 4096 维）
- 语义相似的词在向量空间中距离更近
- 通过训练自动学习获得

示例（简化）：
```
"国王" - "男人" + "女人" ≈ "女王"
```

这就是词嵌入的语义类比特性。

### 语言建模目标

给定一个词序列 `w₁, w₂, ..., wₙ`，语言模型学习联合概率：

P(w₁, w₂, ..., wₙ) = P(w₁) × P(w₂|w₁) × P(w₃|w₁,w₂) × ... × P(wₙ|w₁...wₙ₋₁)

训练目标是最大化**对数似然**：
```
maximize Σ log P(wᵢ | w₁...wᵢ₋₁)
```

换句话说，模型学习根据前面的词，准确预测下一个词。

## Transformer 架构

现在所有主流 LLM 都基于 Transformer 架构（2017年由 Google 提出），核心是**自注意力机制**。

### 整体结构

```
┌─────────────────────────────────────────────────────┐
│                    Input Embedding                  │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                 Positional Encoding                 │
│        (添加位置信息，因为注意力不理解顺序)          │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                  Transformer Block × N              │
│  ┌─────────────────────────────────────────────┐   │
│  │             Multi-Head Attention            │   │
│  └─────────────────────────────────────────────┘   │
│                          ↓                           │
│  │             Add & Layer Norm                   │   │
│                          ↓                           │
│  ┌─────────────────────────────────────────────┐   │
│  │             Feed Forward Network            │   │
│  └─────────────────────────────────────────────┘   │
│                          ↓                           │
│  │             Add & Layer Norm                   │   │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                 Output Layer (Vocab)               │
└─────────────────────────────────────────────────────┘
                          ↓
                   下一个词的概率分布
```

### 自注意力机制（Self-Attention）原理

自注意力让模型在处理每个词时，能够"关注"序列中其他相关的词。

计算步骤：

1. **生成 Q, K, V**
   - Q (Query)：当前词的查询向量
   - K (Key)：每个词的键向量
   - V (Value)：每个词的值向量
   - 通过三个不同的线性变换得到：Q = XWq, K = XWk, V = XWv

2. **计算注意力分数**
   ```
   Attention(Q,K,V) = softmax(QKᵀ / √dₖ) V
   ```
   - QKᵀ：计算 Query 和每个 Key 的相似度
   - √dₖ：缩放，防止数值过大
   - softmax：转换为概率分布，总和为 1
   - 乘以 V：按权重对 Value 加权求和

3. **多头注意力**
   - 将 Q, K, V 分成多个头（head）
   - 每个头学习不同类型的注意力依赖关系
   - 最后将结果拼接起来

例如：
- 一个头关注"它"指代哪个名词
- 另一个头关注形容词和名词的关系
- 第三个头关注语法结构

### 为什么需要位置编码

注意力机制本身是**排列不变**的，它不理解词语的顺序。位置编码给每个位置添加一个特殊的向量，告诉模型词的顺序位置。

常见方法：
- **正弦余弦位置编码**：固定公式生成，GPT-1/2、原始Transformer使用
- **可学习位置编码**：训练过程中学习，BERT、GPT-3 使用
- **旋转位置编码（RoPE）**：近期主流，LLaMA 等使用，外推性好

### 前馈网络（FFN）

每个位置经过注意力层后，通过一个两层的前馈网络：

```
FFN(x) = max(0, xW₁ + b₁)W₂ + b₂
```

- 对每个位置独立处理
- 增加模型的非线性表达能力
- 实际上是两个线性变换加一个ReLU激活

### 残差连接和层归一化

- **残差连接**：帮助深度网络训练，解决梯度消失
  ```
  output = input + attention(input)
  ```
- **层归一化**：稳定训练，加快收敛

## 解码与文本生成

训练好的模型如何生成文本？

### 自回归生成

模型一次生成一个词，将生成的词再作为输入继续生成下一个词：

```
输入: "今天天气真"
模型预测下一个词: "好" → 输出:"今天天气真好"
再把"今天天气真好"作为输入，预测下一个词...
重复直到生成结束符<|endoftext|>
```

### 常用采样策略

**1. 贪婪搜索**
- 每次选概率最大的词
- 缺点：容易重复，陷入循环，结果单调

**2. 采样（Sampling）**
- 根据概率分布随机采样
- 增加多样性，但可能产生不通顺的文本

**3. Top-k 采样**
- 只从概率最高的 k 个词中采样
- k 通常取 40 或 50
- 限制采样范围，提高质量

**4. Nucleus 采样（Top-p）**
- 累积概率超过 p 就停止，从这个集合中采样
- p 通常取 0.9 或 0.95
- 比 Top-k 更灵活，概率分布平坦时采样范围大，尖锐时范围小
- 目前主流方法

**5. 温度采样**
- 用温度 T 调整概率分布：`prob_i = exp(logit_i / T) / Σ exp(logit_j / T)`
- T < 1：分布更尖锐，更确定，更少随机性
- T > 1：分布更平滑，更多样性
- 常用：T = 0.7 ~ 1.0

## 训练过程

### 预训练（Pre-training）

- **数据**：大规模互联网文本（数千亿到数万亿 token）
- **目标**：next token 预测（自监督学习）
- **计算**：需要数千张 A100 GPU 训练数周到数月
- **结果**：模型学到语言规律、知识、推理能力

### 微调（Fine-tuning）

在预训练之后，通常需要进一步微调：

**有监督微调（SFT）**
- 在特定任务的标注数据上微调
- 让模型适应具体任务，比如问答、摘要

**人类反馈强化学习（RLHF）**
- 人类对模型多个输出进行排序
- 训练奖励模型（Reward Model）
- 用 PPO 算法强化学习，对齐人类偏好
- ChatGPT、Claude 都使用这个方法

**对齐（Alignment）**
- 让模型输出符合人类价值观和指令
- 减少有害输出
- 提高帮助性和诚实性

## 扩展法则（Scaling Laws）

研究发现，LLM 的性能符合可预测的扩展规律：

当模型参数、数据量、计算量按比例增加时，**验证损失（perplexity）对数线性下降**。

这意味着：
- 更大的模型 → 更好的性能
- 更多的训练数据 → 更好的性能
- 更多的计算 → 更好的性能

这个规律被经验验证，指导了近年来 LLM 的发展：从 GPT-1 (0.1B) → GPT-3 (175B) → GPT-4 (~1.7T)。

## 上下文窗口

上下文窗口是指模型一次能处理的最大 token 数量：

- GPT-3：2048 / 4096 token
- GPT-3.5：4k / 16k
- GPT-4：8k / 32k / 128k
- LLaMA 2：4k / 128k
- Claude 3：200k

更大的上下文窗口允许模型处理：
- 长文档理解
- 多轮对话
- 大型代码文件

## 涌现能力（Emergent Abilities）

当模型规模超过一定阈值后，突然出现一些小模型没有的能力：

- 链式推理（Chain-of-Thought）
- 少样本学习
- 指令跟随
- 工具使用

这些能力是**在大规模预训练中自发涌现**的，不是被显式编程的。

## 优势与局限性

### 优势

- 强大的泛化能力：一个模型能完成上千种任务
- 零样本/少样本学习：不需要大量标注数据
-  emergent 能力：规模带来意想不到的能力提升
- 作为基座：可以微调适配各种下游任务

### 局限性

- **幻觉问题**：可能编造不存在的事实
- **上下文限制**：无法处理超过窗口的长文本
- **训练成本高昂**：需要巨量计算资源
- **推理能力有限**：复杂逻辑推理容易出错
- **知识截止**：训练后无法自动获取新知识

## 总结

大语言模型的核心原理非常简洁：

1. **架构**：基于 Transformer 解码器，堆叠多层多头自注意力
2. **训练**：在海量文本上进行自监督学习，预测下一个 token
3. **生成**：自回归方式，一次生成一个 token，通过采样策略控制输出
4. **扩展**：性能随参数、数据、计算量可预测地提升

尽管原理简洁，当规模足够大时，模型涌现出惊人的语言理解、知识推理和生成能力，引发了人工智能领域的革命。LLM 技术仍在快速发展中，新的架构和训练方法不断涌现。
