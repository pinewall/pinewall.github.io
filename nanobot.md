# Nanobot 极端模型压缩基本原理

Nanobot 是香港大学数据科学系（HKUDS）发表的关于**极端大语言模型压缩**的研究项目。它致力于探索将数十亿参数的大语言模型压缩到**百万参数级别**（nano-scale），同时尽可能保留原模型的知识和能力。

项目地址：[https://github.com/HKUDS/nanobot](https://github.com/HKUDS/nanobot)

## 研究背景

随着大语言模型的发展，模型参数规模从数百万增长到数万亿：
- GPT-3: 1750 亿参数
- GPT-4: ~1.7 万亿参数
- Llama 2: 70 亿 - 700 亿参数

巨大的模型规模带来了：
- 高昂的推理成本
- 难以部署到端侧设备（手机、嵌入式）
- 巨大的内存占用
- 高能耗

模型压缩研究就是要解决这个问题，**极端压缩**更进一步，目标是压缩到原模型的 **1% 大小甚至更小**。Nanobot 这个名字就体现了这个目标：压缩到"纳米级"的大语言模型。

## 核心压缩原理

### 知识蒸馏（Knowledge Distillation）

知识蒸馏是 Nanobot 这类模型压缩的基础技术：

```
[大模型（教师）]        [小模型（学生）]
   输出概率分布             学习预测
       ↓
  软标签 包含
  更多知识信息
```

原理要点：
- 教师模型（大模型）产生的"软概率"包含类别之间的关系信息
- 学生模型不仅学习真实标签，还要学习教师的输出分布
- 蒸馏损失：`loss = KL(student_logits / T || teacher_logits / T)`
- `T` (温度) 平滑概率分布，放大暗知识信息

优势：
- 成熟稳定，易于实现
- 可以利用无标签数据继续学习

### 结构化剪枝

剪枝通过移除模型中不重要的连接或神经元来减小规模：

**非结构化剪枝**：
- 移除单个权重
- 得到稀疏矩阵
- 需要特殊硬件支持才能加速

**结构化剪枝**（Nanobot 这类实用压缩常用）：
- 移除整个头、层或注意力维度
- 直接得到更小的 dense 模型
- 不需要特殊支持，可直接推理

剪枝流程：
```
1. 训练原模型 → 2. 评估参数重要性 → 3. 剪掉不重要参数 → 4. 微调恢复性能
```

### 量化压缩

量化用更低精度存储权重：

| 精度 | 位数 | 压缩比 |
|------|------|--------|
| FP32 | 32 位 | 1× |
| FP16 | 16 位 | 2× |
| INT8 | 8 位 | 4× |
| INT4 | 4 位 | 8× |

极端压缩中，量化通常和其他方法结合使用，进一步减小体积。

### 神经架构搜索（NAS）

极端压缩往往需要重新设计小模型架构，而不只是压缩现成大模型：

- 搜索空间：宽度、深度、头数、隐藏维数等
- 搜索策略：强化学习、进化算法、随机搜索
- 评价指标：准确率 × 速度 / 大小约束

Nanobot 可能使用 NAS 找到适合 nano-scale 的最优架构。

## 极端压缩的关键技术点

### 1. 保留知识的关键：暗知识转移

大模型学到的不仅仅是预测正确答案，更重要的是学到了**输入之间的关系结构**，这被称为"暗知识"（Dark Knowledge）。通过蒸馏，这些暗知识可以转移给小模型。

### 2. 结构化 vs 非结构化

在极端压缩场景下：
- **结构化压缩**得到完整的小模型，便于部署
- **非结构化压缩**理论压缩率更高，但需要稀疏推理支持
- 实际应用中，结构化更常见

### 3. 分层蒸馏

对于多层 Transformer 模型，可以分层蒸馏：

- 每一层学生对齐教师对应层的输出分布
- 注意力图蒸馏：学生学习教师的注意力模式
- 最终损失结合 logit 蒸馏和中间层蒸馏

### 4. 数据选择

极端压缩需要精心选择训练数据：
- 使用领域特定数据可以获得很好的压缩效果
- 通用任务需要代表性的数据子集
- 可以用原模型过滤高质量数据来蒸馏

## 性能表现

极端压缩到纳米级别后，通常会有：

| 模型规模 | 典型参数量 | 适用场景 | 性能损失 |
|---------|-----------|---------|---------|
| 微模型（Micro） | 10M - 100M | 边缘设备 | 中等 |
| **纳米模型（Nano）** | **1M - 10M** | **极致端侧部署** | **较大但可接受** |
| 皮模型（Pico） | < 1M | 物联网设备 | 显著 |

在特定任务上，经过精心压缩的 5-10M 参数模型可以保留原大模型 90%+ 以上的性能。

## 应用场景

1. **端侧设备**：手机、嵌入式设备运行 LLM
2. **边缘计算**：没有网络连接时本地推理
3. **低延迟服务**：不需要云端往返，毫秒级响应
4. **多实例部署**：大量并发请求降低成本
5. **隐私保护**：数据不出本地，保护用户隐私

## 挑战与权衡

极端压缩面临的核心挑战：

1. **能力-大小权衡**：压缩越小，性能损失越大，需要找到应用可接受的平衡点
2. **泛化性**：压缩后的模型在分布外数据上性能下降更快
3. **训练成本**：虽然推理小，但压缩搜索过程本身可能很昂贵
4. **任务特异性**：极端压缩通常对任务敏感，通用极端压缩仍很困难

## 总结

Nanobot 作为极端语言模型压缩项目，其核心思想是：**通过知识蒸馏、结构化剪枝和量化等技术组合，将大语言模型压缩到纳米（百万）参数级别，使其能够部署在资源受限的端侧设备上**。

随着端侧AI需求的增长，极端模型压缩越来越受到关注。Nanobot 这类研究推动了大语言模型从云端走向边缘，使得更多设备能够运行人工智能语言能力。
