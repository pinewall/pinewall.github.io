# OpenClaw 个人AI助手基本原理

OpenClaw 是一个**全功能个人 AI 助手**，官网：[https://openclaw.ai/](https://openclaw.ai/)，定位是 *"The AI that actually does things. Your personal assistant on any platform"* —— 能真正做事的 AI，在任何平台都可以使用。

OpenClaw 基于 Nanobot 内核构建，Nanobot 提供 AI Agent 核心引擎，OpenClaw 提供完整的用户体验和产品化封装。

## 核心定位

OpenClaw 的设计目标：

- ✅ **真正能干活**：不仅仅聊天，能实际操作你的文件、执行命令、完成任务
- ✅ **跨平台**：支持飞书、Telegram 等多个 IM 渠道
- ✅ **自托管**：你可以完全掌控自己的 AI 助手
- ✅ **长期记忆**：自动记忆你说过的话，持续学习你的偏好
- ✅ **工具使用**：能读写文件、执行命令、搜索网页
- ✅ **隐私可控**：数据留在你自己的服务器上

## 核心能力

### 1. 工具使用

OpenClaw 继承了 Nanobot 的工具能力，可以：

| 工具 | 能力 |
|------|------|
| 文件操作 | 读取、写入、编辑本地项目文件 |
| Shell 执行 | 在你的服务器上执行命令 |
| 网络搜索 | 搜索最新信息 |
| 网页抓取 | 获取网页内容 |
| 定时任务 | 设置定时提醒和任务 |
| 子代理生成 | 派生子任务并行处理 |

这让 OpenClaw **不仅仅是聊天**，可以真正帮你完成实际工作。

### 2. 长期记忆系统

OpenClaw 使用 Nanobot 的两层记忆架构：

- **长期记忆 MEMORY.md**：存储你的偏好、项目上下文、关键决策
- **历史日志 HISTORY.md**：按时间线记录对话摘要，可搜索

当对话太长时，**自动压缩**旧对话：
1. 调用 LLM 提炼关键信息更新长期记忆
2. 摘要写入历史日志
3. 只有最近未压缩的对话放在上下文窗口
4. 长期记忆注入到 system prompt，LLM 总能看到

这样即使对话几个月，也不会超出上下文窗口限制。

### 3. 多渠道支持

OpenClaw 可以连接到你日常使用的即时通讯软件：

- **飞书**：企业/团队常用
- **Telegram**：海外用户常用
- **本地 CLI**：开发调试

你不需要打开新的 App 或网站，在你已经在使用的 IM 中就能使用 AI 助手。

### 4. 多 LLM 提供商支持

支持几乎所有主流 LLM 提供商：

- Anthropic Claude
- OpenAI GPT
- OpenRouter（聚合 30+ 模型）
- 本地 Ollama
- DeepSeek
- 通义千问
- 文心一言
- ... 等等 30+ 提供商

你可以自由切换模型，选择性价比最高的。

## 部署方式

OpenClaw 设计为**自托管**：

1. **你自己的服务器**上运行
2. **你自己保管** API 密钥
3. **数据存在你这边**，对话不离开你的部署
4. 适合注重隐私的用户

部署步骤：
```bash
# 安装
pip install openclaw

# 初始化
openclaw onboard

# 配置 API 密钥和渠道
# ~/.openclaw/config.yaml

# 启动
openclaw gateway
```

启动后，OpenClaw 就会连接到你配置的 IM 渠道，你就可以开始对话了。
